# Scalar IST デモ録画シナリオ

## 録画方法

macOS: `Shift + Cmd + 5` でスクリーンレコーディング開始

## 事前準備

```bash
# APIサーバー起動（ターミナル1）
cd ~/scalar-ist-app/app
./gradlew run -PmainClass=com.scalar.ist.app.ApiServer \
  --args="/Users/kanekomasato/scalar-ist-app/config/controller.properties"

# フロントエンド起動（ターミナル2）
cd ~/scalar-ist-app/web
npm run dev
```

## シナリオ（約3分）

### Scene 1: トップページ（10秒）
1. http://localhost:3001 を開く
2. 「Scalar IST Demo」のタイトルと説明を見せる
3. **「デモを開始する」** をクリック
4. 2つのシーンリンクが表示される

### Scene 2: サービス登録の同意画面（60秒）
1. **「サービス登録時の個人情報取得同意」** リンクをクリック
2. 同意画面が表示される — ヘッダー（青い帯）を見せる
3. 必須2項目（✓済み）と任意3項目を確認
4. **「サービス提供のための個人情報利用」の ▶ をクリック** → 展開
   - 📋 データ項目: 氏名、メールアドレス、電話番号、IPアドレス
   - 🎯 利用目的: アカウント作成・本人確認・不正検知
   - ⏱ 利用期限: サービス退会後1年間
   - 🎁 便益: サービスの安定的な提供
5. **「広告パートナーへのデータ提供」の ▶ をクリック** → 展開
   - 第三者提供先: 広告パートナー A社・B社 を確認
6. **「マーケティングメール配信」** にチェックを入れる（任意項目の選択）
7. **「同意する」** をクリック

### Scene 3: 確認画面（15秒）
1. 確認画面が表示される
   - ✓ サービス提供（データ: 氏名, メール, 電話, IP）
   - ✓ カスタマーサポート
   - ✓ マーケティングメール
   - − アクセス解析（未選択）
   - − 広告パートナー（未選択）
2. 「ScalarDLの改ざん検知可能な台帳に記録されます」の注意書きを見せる
3. **「同意して送信」** をクリック

### Scene 4: 完了画面（15秒）
1. ✅ 「回答が記録されました」が表示
2. 受付番号、記録日時、記録先（ScalarDL改ざん検知台帳）、同意項目（3/5）を確認

### Scene 5: 管理ダッシュボード（40秒）
1. http://localhost:3001/admin を開く（または「管理ダッシュボード」リンク）
2. 統計カード: 文書数、下書き、公開中、同意済み、拒否
3. **同意文書一覧テーブル**: タイトル、ステータス（published/draft/inactive）、同意状態（approved/rejected）、作成日時
4. 最新の「サービス登録時の個人情報取得同意」が **approved** になっていることを確認
5. 下にスクロール → **同意記録テーブル**: 同意文書ID、データ主体、状態、記録日時
6. approved / rejected の記録が時系列で確認できることを見せる
7. フッター「Powered by ScalarDL 3.12.2 + ScalarDB 3.17.1 + PostgreSQL 15」

### Scene 6:（オプション）NDAシーン（30秒）
1. トップに戻り、「業務委託先NDA」リンクをクリック
2. 全3項目が**必須**（赤いバッジ）
3. 各項目を展開して NDA条文を確認
4. **「確認・承諾する」** をクリック → 完了

## ナレーション例

> Scalar ISTは、ScalarDLの改ざん検知可能な台帳を使って、同意記録を安全に保全するテンプレートです。
>
> まず、事業者がデータセット単位で同意文書を作成します。各データセットには利用目的、取得するデータ項目、第三者提供先、利用期限、便益が含まれます。
>
> ユーザーは招待リンクを開くだけで、各項目の詳細を確認し、必須項目と任意項目を選択して同意できます。
>
> 同意記録はScalarDLの分散台帳に記録され、「誰が」「どの同意文書に」「どのような形で同意したか」が改ざん検知可能な形で保全されます。
>
> 管理画面では、全同意文書の状態と同意記録の履歴を一覧で確認できます。
